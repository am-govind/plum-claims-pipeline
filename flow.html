    U->>API: POST /api/claims
(member, category, amount, docs)
    API->>P: run_pipeline(ClaimState)

    P->>I: intake.run(state)
    alt member not found OR policy mismatch
        I-->>P: state.early_stop = true
        P-->>API: state (no decision)
        API->>DB: save(state)
        API-->>U: 200 NEEDS_CORRECTION
+ user_message
    else valid
        I-->>P: agent_results.intake = OK (1.0)
    end

    P->>DV: document_verification.run(state)
    alt missing type / UNREADABLE / patient mismatch
        DV-->>P: early_stop + typed error
        P-->>API: state (no decision)
        API->>DB: save(state)
        API-->>U: 200 NEEDS_REUPLOAD
+ actionable message
    else all checks pass
        DV-->>P: agent_results.document_verification = OK
    end

    P->>EX: extraction.run(state)
    par per-document concurrent extraction (asyncio.gather)
        EX->>LLM: extract_document(doc, hint_category)
        LLM-->>EX: ExtractedDocument + LLMUsage
(tokens, latency, USD)
    end
    EX->>V: validate_extraction(doc)
    V-->>EX: validation_issues
(bill total / date / regex)

    alt low confidence (<0.7) OR validator issues
        EX-->>P: state.extracted (with issues)
        P->>P: re_verification (cap=1)
        P->>EX: re-run extraction
        EX-->>P: refreshed state.extracted
    else clean
        EX-->>P: state.extracted = clean
    end

    P->>CD: contradiction_detection.run(state)
    Note over CD: 5 checks: patient name, dates,
hospital name, amount reconcile,
diagnosis-treatment compatibility
    CD-->>P: state.contradictions

    P->>PA: policy_adjudication.run(state)
    PA->>RE: engine.evaluate(state)
    RE-->>PA: RuleResult[]
(coverage, waiting periods,
exclusions, pre-auth, prescription)
    PA-->>P: state.findings extended

    P->>FC: financial_calculation.run(state)
    Note over FC: Deterministic 6-step:
line-item exclusions → per-claim cap →
network discount → co-pay →
sub-limit → YTD remaining
    FC-->>P: state.line_decisions
+ FINANCIAL_CALCULATION finding

    P->>FD: fraud_detection.run(state)
    alt simulate_component_failure (TC011)
        FD-)P: raise SimulatedComponentFailure
        Note over P: orchestrator catches:
state.degraded = true,
confidence delta -0.25,
failed_components += "fraud_detection"
    else normal
        FD-->>P: state.fraud_signals
(same-day / monthly / high-value)
    end

    alt fraud signals AND no policy rule failed
        P->>P: policy_reconsider (cap=1)
        Note over P: flag for manual review
because fraud disagrees with policy
    end

    P->>DS: decision_synthesizer.run(state)
    DS->>DS: compute_confidence
Σ wᵢ·Cᵢ − α·contradiction − β·degraded
    DS->>DS: route by precedence:
HARD_REJECT → FRAUD_INVESTIGATION →
ESCALATED_MEDICAL_REVIEW → MANUAL_REVIEW →
NEEDS_CLARIFICATION → PARTIAL → APPROVED → REJECTED
    DS->>DS: build_explanation_tree(state, decision)
    DS-->>P: state.decision
(status, approved_amount, reasons,
confidence, user_message, breakdown,
cost rollup)

    P-->>API: ClaimState (final, with trace)
    API->>DB: ClaimsRepository.save(state)
    DB-->>API: ok
    API-->>U: 200 ClaimResponse
(claim_id, full state)
Diagram reflects backend/app/graph/pipeline.py + the agents in backend/app/agents/ as currently wired. Critical agents (intake, document_verification, decision_synthesizer) re-raise; non-critical agents are caught by the orchestrator wrapper, which records the failure on the trace and continues in degraded mode.
